**Elapsed time formatting:**

private func formatElapsed(_ seconds: Int64) -> String {
    let h = seconds / 3600
    let m = (seconds % 3600) / 60
    let s = seconds % 60
    if h > 0 {
        return String(format: "%d:%02d:%02d", h, m, s)
    }
    return String(format: "%d:%02d", m, s)
}

4. **Active session resume check (D-14, WORK-09) -- CRITICAL WIRING:**

The resume prompt MUST be triggered by observing `workoutViewModel.hasActiveSessionFlow`, NOT by unchecked local state. This is the fix for the crash recovery UI trigger mechanism.

Add a dedicated async observation function:
```swift
private func observeHasActiveSession() async {
    do {
        for try await hasActive in asyncSequence(for: workoutViewModel.hasActiveSessionFlow) {
            if hasActive.boolValue {
                showResumePrompt = true
            }
        }
    } catch {
        print("Error observing hasActiveSession: \(error)")
    }
}
```

Wire it into the view lifecycle alongside the existing template observation. Update the `.task` modifier to observe both flows:
```swift
.task {
    await withTaskGroup(of: Void.self) { group in
        group.addTask { await observeTemplates() }
        group.addTask { await observeHasActiveSession() }
    }
}
.onAppear {
    workoutViewModel.checkForActiveSession()
}
```

Flow: `onAppear` calls `checkForActiveSession()` which queries Room and sets `_hasActiveSession.value = true/false` in the ViewModel. The `observeHasActiveSession()` async function receives the emission via `asyncSequence(for: workoutViewModel.hasActiveSessionFlow)`. When it emits `true`, it sets `showResumePrompt = true`, which presents the alert.

    **Add `reorderExercise()` public method** (per D-01, D-02, FLOW-03):
    ```kotlin
    fun reorderExercise(from: Int, to: Int) {
        val active = _sessionState.value as? WorkoutSessionState.Active ?: return
        val pendingStart = active.currentExerciseIndex + 1
        val absFrom = pendingStart + from
        val absTo = if (to > from) pendingStart + to - 1 else pendingStart + to
        if (absFrom < pendingStart || absFrom >= active.exercises.size) return
        if (absTo < pendingStart || absTo >= active.exercises.size) return

        val list = active.exercises.toMutableList()
        val item = list.removeAt(absFrom)
        list.add(absTo, item)

        // Reorder templateOriginalIndices in parallel
        val idxItem = templateOriginalIndices.removeAt(absFrom)
        templateOriginalIndices.add(absTo, idxItem)

        _sessionState.value = active.copy(exercises = list)
        persistExerciseOrder()
    }
    ```

    Note on SwiftUI `.onMove` index semantics: SwiftUI passes `from` as the source index within the pending sublist and `to` as the destination insertion point. When moving an item DOWN (to > from), the destination index is one higher than the final position because the source item hasn't been removed yet. The `absTo` calculation with `if (to > from) ... - 1 else ...` compensates for this. Alternatively, use the simpler pattern of operating on a mutable list copy using Swift's `Array.move(fromOffsets:toOffset:)` equivalent -- but since we call from Swift with the raw indices, this adjustment is needed.

    ACTUALLY -- simpler approach: match the Swift `.onMove` semantics directly. SwiftUI `.onMove` gives `IndexSet` source and `Int` destination which are designed for `Array.move(fromOffsets:toOffset:)`. The Kotlin ViewModel should accept these same semantics. Use this implementation instead:

    ```kotlin
    fun reorderExercise(from: Int, to: Int) {
        val active = _sessionState.value as? WorkoutSessionState.Active ?: return
        val pendingStart = active.currentExerciseIndex + 1
        // from and to are relative to the pending sublist, using move(fromOffset, toOffset) semantics
        val absFrom = pendingStart + from
        val absTo = pendingStart + to
        if (absFrom < pendingStart || absFrom >= active.exercises.size) return
        if (absTo < pendingStart || absTo > active.exercises.size) return  // toOffset can equal size (append)

        val list = active.exercises.toMutableList()
        val item = list.removeAt(absFrom)
        // After removal, if absTo > absFrom, the effective insertion index is absTo - 1
        val insertAt = if (absTo > absFrom) absTo - 1 else absTo
        list.add(insertAt, item)

        val idxItem = templateOriginalIndices.removeAt(absFrom)
        templateOriginalIndices.add(insertAt, idxItem)

        _sessionState.value = active.copy(exercises = list)
        persistExerciseOrder()
    }
    ```

## Tech Debt

### Phase 10: Color & Accessibility

1. **14 hardcoded accent color instances in non-workout views** — Files: MainTabView, WorkoutEmptyStateView, TemplateListView, ExercisePickerView, ExerciseCatalogView, ExerciseDetailView, CreateExerciseView, AnatomyPickerView, AnatomyBackShape, AnatomyFrontShape. Phase 10 intentionally scoped migration to workout views only. Non-blocking but should be migrated for full consistency.

2. **PB label abbreviation not VoiceOver-friendly** — `Text("PB: 50.0 kg")` reads literally as "PB: fifty point zero kg" instead of "Personal best: fifty kilograms." The label is readable by default Text behavior, but the abbreviation is suboptimal for screen readers. Minor.

### Phase 09: Sequential Coroutine Ordering

3. **Save & Exit relies on Main.immediate implicit ordering** — `enterReview()` and `saveReviewedWorkout()` are called sequentially from Swift but each launches a separate coroutine in Kotlin. Works because `enterReview()` has no suspension points. If a future change adds a suspend call inside `enterReview()`, the ordering breaks silently. Consider combining into a single `saveAndExit()` ViewModel method for robustness.

**Total: 3 items across 2 phases. None are blockers.**

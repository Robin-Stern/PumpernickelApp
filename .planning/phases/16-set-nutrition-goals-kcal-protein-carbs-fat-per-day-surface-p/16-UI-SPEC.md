# Phase 16 — UI Design Contract

> Visual and interaction contract for Phase 16: Set nutrition goals (kcal/protein/carbs/fat per day).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — KMP + Compose Material 3 + SwiftUI project) |
| Preset | not applicable |
| Component library (Android) | Jetpack Compose Material 3 (dynamic accent via `PumpernickelTheme`) |
| Component library (iOS) | SwiftUI native components |

| Icon library (iOS) | SF Symbols (system images) |

**Design system note:** No new design system tooling is introduced. This phase reuses every established token, component, and pattern from the existing codebase. `PumpernickelTheme` already provides dynamic color per D-post-v1.5 (8 accent presets + light/dark mode). All color references below use M3 semantic roles, not hard-coded hex values, so they adapt automatically.

## Spacing Scale

Declared values (multiples of 4, matching the existing codebase):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4dp/pt | Icon gaps, label-to-picker gap, legend dot-to-label gap |
| sm | 8dp/pt | Between banner icon and text; between stat field and its label; top padding after TopAppBar |
| md | 16dp/pt | Section card internal padding; horizontal screen padding; stat row padding |

| 3xl | 64dp/pt | Not used |

Exceptions:
- DrumPicker item height: **40dp** (existing `DrumPicker.kt` constant — do not change)

- Suggestion card internal padding: **12dp** (tighter than full card to pack three cards side-by-side or in a row)
- Touch target minimum: **44pt** (iOS) / **48dp** (Android) for all interactive controls (banner dismiss "×", stat picker controls, Save button)

All type roles map to the M3 type scale on Android and the equivalent SwiftUI `.font()` modifier on iOS.

| Role | M3 Android token | iOS SwiftUI font | Weight | Line Height | Usage in this phase |

## Drum Picker Ranges (Claude's Discretion — D-16-03)

Per-macro ranges for the five `DrumPicker` instances in the editor. Values are whole integers.

| Macro | Min | Max | Step | Default (NutritionGoals) |
|-------|-----|-----|------|--------------------------|
| kcal | 800 | 6 000 | 50 | 2 500 |
| Protein (g) | 20 | 400 | 5 | 150 |
| Carbs (g) | 20 | 700 | 5 | 300 |
| Fat (g) | 10 | 250 | 5 | 80 |
| Sugar (g) | 0 | 200 | 5 | 50 |

**Step rationale:** 50 kcal steps keep the kcal list length at 105 items (manageable scroll). 5 g steps for macros keep lists at max 76 items. Both match the snappy iOS Picker(.wheel) feel.

**Gram rounding for suggestion cards (Claude's Discretion):** Macro splits derived from TDEE are rounded to the nearest 5 g before being pushed into the drum pickers. Exception: values < 20 g are rounded to the nearest 1 g to avoid rounding to 0. This prevents the picker from jumping to a non-existent step.

## Suggestion Card "Selected" Visual Treatment (Claude's Discretion)

Three cards: Cut / Beibehalten / Bulk (displayed in a horizontal row on wide screens; vertical stack on narrow).

| State | Android | iOS |
|-------|---------|-----|
| Unselected | `ElevatedCard` or `OutlinedCard` with `surfaceVariant` background, no border | `RoundedRectangle` with `secondarySystemGroupedBackground` fill, 1pt `Color(.separator)` stroke |
| Selected | `OutlinedCard` with `colorScheme.primary` 2dp border + `colorScheme.primary.copy(alpha = 0.08f)` background | `RoundedRectangle` with `Color.appAccent` 2pt stroke + `Color.appAccent.opacity(0.08)` fill |
| Hover / pressed | M3 ripple on the card surface | SwiftUI `.buttonStyle(.plain)` with opacity press animation |

Card content layout (same on both platforms):
1. Title badge: "Cut" / "Beibehalten" / "Bulk" — `titleMedium` weight SemiBold
2. Kcal value: e.g. "2 100 kcal" — `titleMedium` weight Regular, colored with `CalorieRingColor`
3. Macro breakdown: three lines: "Protein 156 g", "Kohlenhydrate 210 g", "Fett 58 g" — `labelSmall` + macro ring color dot (8dp circle) per line
4. Sugar line: "Zucker 50 g" — `labelSmall` + sugar ring color dot

Tapping a card selects it (only one can be selected at a time), fills the pickers below, and visually highlights. Tapping the same card again has no effect (does not deselect — the user must pick one of the three).

## Overview Banner (Claude's Discretion — D-16-13 / D-16-14)

**Placement:** Above the NutritionRingsCard in the Overview scroll column, inserted before it in the `Column`/`VStack`.

**Visibility:** Only shown while `nutritionGoalsBannerVisible == true` in the ViewModel.

**Copy:**
- Leading icon: SF Symbol `target` (iOS) / `Icons.Outlined.TrackChanges` (Android) — in `colorScheme.primary` / `Color.appAccent`
- Main text: **"Persönliche Ziele setzen"** — `bodyMedium` / `.body` weight Regular

- Trailing: chevron "→" icon in accent color + dismiss "×" button

**Dismiss animation:**
- Android: `AnimatedVisibility` with `slideOutVertically` + `fadeOut`, duration 300ms

**Banner card style:**
- Android: `Card` with `RoundedCornerShape(12.dp)`, `colorScheme.surfaceVariant.copy(alpha = 0.6f)` background
- iOS: `RoundedRectangle(cornerRadius: 12)` with `Color(.secondarySystemGroupedBackground)` fill

## Screen Layout: NutritionGoalsEditorScreen / NutritionGoalsEditorView

### Entry

- **Android:** `NutritionGoalsEditorScreen` pushed onto the Overview-tab NavHost via `navController.navigate(NutritionGoalsEditorRoute)`. Back navigation = standard back button in TopAppBar.
- **iOS:** `NutritionGoalsEditorView` presented as a `.sheet` from the edit button on `OverviewView`. Dismiss via `.presentationDragIndicator(.visible)` + swipe down or "Abbrechen" button.

### Structure (single scrollable screen with 3 sections — D-16-02)

```
Screen
├── TopAppBar / NavigationBar title: "Ernährungsziele"
│   └── [iOS] "Abbrechen" leading button (dismisses sheet without saving)
│
├── ScrollView / LazyColumn
│   │
│   ├── Section 1: "Meine Stats" (collapsible — D-16-09)
│   │   ├── Disclosure group header / ExpandableCard toggle: "Meine Stats"
│   │   └── [when expanded]
│   │       ├── Weight field: NumericField or DrumPicker-style row — "Gewicht (kg)"
│   │       ├── Height field: "Körpergröße (cm)"
│   │       ├── Age field: "Alter"
│   │       ├── Sex selector: SegmentedControl "Männlich / Weiblich"
│   │       └── Activity tier: DropdownMenu (Android ExposedDropdownMenuBox) /
│   │                           Picker(.menu) (iOS) — German labels from D-16-05
│   │
│   ├── Section 2: "Vorschlag berechnen" (always visible)
│   │   └── HStack / Row of 3 tappable SuggestionCards: Cut · Beibehalten · Bulk
│   │       (live-updating as stats change; tapping pre-fills Section 3 pickers)
│   │
│   └── Section 3: "Zielwerte anpassen"
│       ├── DrumPickerRow: "Kalorien" — range 800–6000, step 50, unit "kcal"
│       ├── DrumPickerRow: "Protein" — range 20–400, step 5, unit "g"
│       ├── DrumPickerRow: "Kohlenhydrate" — range 20–700, step 5, unit "g"
│       ├── DrumPickerRow: "Fett" — range 10–250, step 5, unit "g"
│       └── DrumPickerRow: "Zucker" — range 0–200, step 5, unit "g"
│
└── Sticky bottom bar (or last item in scroll)
    └── Save button: "Ziele speichern" — full-width, accent-filled
```

### Stat input fields

- **Android:** `OutlinedTextField` with numeric keyboard (`keyboardType = KeyboardType.Number`). One field per stat (weight, height, age). No drum picker for stats inputs — text fields are more ergonomic for arbitrary numeric entry.
- **iOS:** SwiftUI `TextField` with `.keyboardType(.numberPad)`. Same layout.
- Weight field shows "kg" suffix inline (not as a separate label).
- Height field shows "cm" suffix inline.

### Section collapse behavior (D-16-09)

- Stats section starts **expanded** on first open (when `userPhysicalStats == null`); starts **collapsed** on subsequent opens (when stats already stored).
- Android: `AnimatedVisibility` wrapping the stats content, toggled by a header `Row` with a `chevron_up / chevron_down` icon.
- iOS: `DisclosureGroup` with the section title as the label.

## Interaction States

| State | Trigger | Visual Response |
|-------|---------|----------------|
| Stats not yet entered | `userPhysicalStats == null` | Stats section expanded; suggestion cards show dashes "—" for macro values |
| Stats changing (any field edit) | User types in stat field | Suggestion cards update live via ViewModel recomputation; no loading indicator needed (pure math) |
| Suggestion card tapped | User taps Cut / Beibehalten / Bulk | Card highlights (selected style); drum pickers animate to new values via `LaunchedEffect(selectedItem)` scroll in DrumPicker |
| Picker scrolling | User flings DrumPicker | Existing snap behavior (unchanged); selected suggestion card visually deselects if picker value diverges from card values by more than ±1 step |
| Save tapped | User taps "Ziele speichern" | Button shows `CircularProgressIndicator` inset for 300ms (matches DataStore write latency); screen dismisses on success |
| Banner tapped | User taps the banner body | Navigate to editor (same as edit button); banner persists until Save or explicit dismiss |
| Banner dismissed | User taps "×" | Slide-out animation (see above); `dismissBanner()` call; never shown again |
| Goals saved | `setNutritionGoals()` completes | Banner dismissed (D-16-14); screen popped; no toast/snack needed (Overview rings update immediately via Flow) |

## Platform-Specific Notes

### Android
- Editor screen is a regular Compose `Screen` navigated via the Overview-tab NavHost (`navController.navigate(NutritionGoalsEditorRoute)`).
- Stats section collapse uses `AnimatedVisibility(visible = isStatsExpanded)`.
- Drum pickers are laid out in a `LazyColumn` (scrollable editor) — each picker needs a fixed height of `200.dp` (5 visible items × 40dp) to prevent LazyColumn measurement conflicts. Set `modifier = Modifier.height(200.dp)` on each `DrumPicker`.
- The three suggestion cards use `Row(horizontalArrangement = Arrangement.spacedBy(8.dp))` — each card gets `Modifier.weight(1f)`.

### iOS
- Editor presented as `.sheet(isPresented: $showEditor)` from a `@State` variable on `OverviewView`.
- Stats section collapse uses `DisclosureGroup("Meine Stats", isExpanded: $statsExpanded)`.
- Drum pickers: native SwiftUI `Picker(.wheel)` — five pickers in a `Form` / `Section` or a custom `HStack` layout. Use `MultiComponentPicker` pattern from `WorkoutSessionView.swift` if side-by-side pickers are needed; otherwise stack them vertically in a `Form`.
- Suggestion cards: `HStack(spacing: 8)` with `frame(maxWidth: .infinity)` on each card.

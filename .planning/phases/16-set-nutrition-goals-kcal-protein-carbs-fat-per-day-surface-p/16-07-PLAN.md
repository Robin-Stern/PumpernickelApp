<objective>
Close Gap 3 (WR-05) at the shared root cause: snap kcal to the nearest multiple of 50
inside `TdeeCalculator.buildSplit()` so suggestion-applied kcal values are always
present in the picker ranges on both platforms (Android DrumPicker `(800..6000 step 50)`,
iOS `stride(from: 800, through: 6000, by: 50)`).

Today `buildSplit()` returns a raw `roundToInt()` result (e.g. 2259 for the reference
80kg/180cm/30/male moderately-active cut suggestion). The picker silently ignores the
out-of-range selection: Android DrumPicker's `LaunchedEffect(selectedItem)` does
`items.indexOf(2259)` which returns -1 and skips the scroll, leaving the wheel at the
previous position. iOS Picker(.wheel) has undefined behavior with a non-matching tag.

The fix MUST be in the shared layer (no platform-side change needed) so both platforms
benefit from one source of truth. Macros (protein, fat, carbs) MUST be derived from the
SNAPPED kcal so the card display ("2250 kcal · 175 g protein · …") and the picker
state remain consistent.

Purpose: Suggestion cards become functional. Pickers visibly scroll on tap.
Output: TdeeCalculator.kt with snap-to-50 + 2 new tests proving the snap.
</objective>

# Phase 1 -- UI Design Contract

> Visual and interaction contract for the Foundation & Exercise Catalog phase.
> Platform: SwiftUI (iOS, primary) + Compose (Android). iOS-first development.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native mobile -- no web design system) |
| Preset | not applicable |
| Component library | SwiftUI native (iOS) / Material 3 via Compose (Android) |
| Icon library | SF Symbols (iOS) / Material Icons (Android) |
| Font | System default -- San Francisco (iOS) / Roboto (Android) |

**Rationale:** Per D-01, the app uses platform-native UI. Each platform follows its own HIG/Material guidelines. The design contract below specifies logical values that map to platform conventions. SwiftUI and Compose both have semantic spacing/typography systems; the values below are the source of truth that each platform implementation must respect.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding within chips |
| sm | 8px | Compact element spacing, chip internal padding, search bar icon inset |
| md | 16px | Default element spacing, list item horizontal padding, card content padding |
| lg | 24px | Section padding, spacing between filter chip row and list |
| xl | 32px | Screen-level horizontal padding, spacing between major sections |
| 2xl | 48px | Empty state illustration to text gap |
| 3xl | 64px | Top padding for empty state content centering |

Exceptions:
- Bottom navigation bar height: 49pt (iOS standard) / 80dp (Material 3 standard)
- FAB: 56dp diameter (Material 3 standard) on Android; on iOS use a circular button at 56pt
- Touch targets: minimum 44pt (iOS) / 48dp (Android) for all tappable elements
- Anatomy SVG viewer: full-width with 16px horizontal margin, aspect ratio preserved from source SVG

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (16sp / 16pt) | Regular (400) | 1.5 (24px) | Exercise instructions, detail screen text, empty state body |
| Label | 14px (14sp / 14pt) | Medium (500) | 1.4 (20px) | Filter chips, metadata labels (equipment, level, force), subtitle in list rows |
| Heading | 20px (20sp / 20pt) | Semibold (600) | 1.2 (24px) | Screen titles, exercise name on detail screen, section headers |
| Display | 28px (28sp / 28pt) | Bold (700) | 1.2 (34px) | Not used in Phase 1 (reserved for workout session numbers in Phase 3) |

**Platform mapping:**
- iOS (SwiftUI): Body = `.body`, Label = `.subheadline`, Heading = `.title3`, Display = `.title1`. Override sizes if platform defaults deviate from values above by more than 2px.
- Android (Compose): Body = `bodyLarge`, Label = `labelLarge`, Heading = `titleMedium`, Display = `headlineMedium`.

## Color

The app uses a dark theme as the default (gym/fitness convention -- users often train in dim environments).

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `#FFFFFF` | `#121212` | Screen background, primary surface |
| Secondary (30%) | `#F5F5F5` | `#1E1E1E` | Cards, list item backgrounds, bottom nav bar, search bar fill |
| Accent (10%) | `#4CAF50` (green) | `#66BB6A` (green) | See reserved list below |
| Destructive | `#D32F2F` | `#EF5350` | Delete custom exercise (future), destructive confirmations |

**Accent reserved for (exhaustive list -- never used elsewhere):**
1. FAB (+) button background on exercise catalog screen
2. Primary CTA button on Workout tab empty state ("Browse Exercises")
3. Active/selected muscle region fill on anatomy SVG
4. Active bottom navigation tab icon tint
5. Search bar cursor / text selection highlight

**Muscle region colors (anatomy SVG specific):**
| State | Color (Dark Mode) | Color (Light Mode) |
|-------|-------------------|-------------------|
| Unselected / idle | `#3A3A3A` | `#D0D0D0` |
| Hover / pressed | `#4A4A4A` | `#B0B0B0` |
| Selected (active filter) | `#66BB6A` (accent) | `#4CAF50` (accent) |
| Body outline / stroke | `#2A2A2A` | `#E0E0E0` |

**Platform mapping:**
- iOS: Use SwiftUI `Color` with adaptive light/dark values via `Color(uiColor:)` or asset catalog.
- Android: Use Material 3 `MaterialTheme.colorScheme` with `darkColorScheme()` / `lightColorScheme()`.
- Both platforms must support system-level dark/light mode switching.

## Interaction Contracts

### Navigation Flow

```
App Launch
  --> Bottom Nav (Workout tab active)
    --> Workout Tab Empty State
      --> [Browse Exercises] --> Exercise Catalog
        --> [Tap row] --> Exercise Detail --> [Back] --> Exercise Catalog
        --> [FAB +] --> Create Exercise --> [Create] --> Exercise Catalog
        --> [Body icon in chips] --> Anatomy Picker (sheet) --> [Select] --> Filter applied
    --> Overview Tab --> Placeholder
    --> Nutrition Tab --> Placeholder
```

### Transitions

| From | To | Transition |
|------|----|-----------|
| Tab to Tab | Bottom nav tabs | Cross-fade (no slide), instant |
| Workout Empty -> Exercise Catalog | Push navigation | Slide from right (iOS) / shared-axis forward (Android) |
| Exercise Catalog -> Exercise Detail | Push navigation | Slide from right (iOS) / shared-axis forward (Android) |
| Exercise Catalog -> Create Exercise | Push navigation | Slide from right (iOS) / shared-axis forward (Android) |
| Any -> Anatomy Picker | Bottom sheet | Slide up from bottom with dimmed backdrop |
| Create Exercise -> Exercise Catalog (after save) | Pop navigation | Slide to right (iOS) / shared-axis backward (Android) |

### Touch Feedback

| Element | Feedback |
|---------|----------|
| Exercise list row | iOS: standard highlight. Android: ripple effect |
| Filter chip | Immediate color change (selected/unselected swap) |
| Anatomy SVG region | Immediate fill color change to accent on tap |
| FAB | iOS: scale-down 0.95 on press. Android: elevation change + ripple |
| CTA buttons | iOS: opacity 0.7 on press. Android: ripple |
| Bottom nav tab | Icon tint change, no animation delay |

### Keyboard Behavior

| Screen | Behavior |
|--------|----------|
| Exercise Catalog (search) | Keyboard appears on search bar focus. List scrolls above keyboard. "Search" return key. Dismiss on scroll down or tap outside. |
| Create Exercise (name field) | Keyboard appears on field focus. Form scrolls to keep active field visible. "Next" return key advances to next field. "Done" on last field dismisses keyboard. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Minimum touch target | 44pt (iOS) / 48dp (Android) for all interactive elements |
| Anatomy SVG regions | Each region must have an accessibility label matching its muscle group name (e.g., "Chest", "Shoulders") |
| Color contrast | All text must meet WCAG AA (4.5:1 for body text, 3:1 for large text) against its background in both light and dark modes |
| Screen reader order | Exercise list: name first, then muscle group. Detail screen: logical top-to-bottom reading order. |
| Dynamic type | iOS: support Dynamic Type scaling. Android: respect system font size. Typography values above are base sizes. |
| Motion | Respect "Reduce Motion" (iOS) / "Remove animations" (Android) system settings for transitions |

sealed class WorkoutSessionState {

}

**Private helper methods:**

`startRestTimer(durationSeconds: Int)` -- per D-04, D-05:
```kotlin
private fun startRestTimer(durationSeconds: Int) {
    timerJob?.cancel()
    timerJob = viewModelScope.launch {
        val startTime = Clock.System.now().toEpochMilliseconds()
        var remaining = durationSeconds
        updateRestState(RestState.Resting(remaining, durationSeconds))
        while (remaining > 0) {
            delay(1000L)
            remaining = durationSeconds - ((Clock.System.now().toEpochMilliseconds() - startTime) / 1000).toInt()
            if (remaining < 0) remaining = 0
            updateRestState(RestState.Resting(remaining, durationSeconds))
        }
        updateRestState(RestState.RestComplete)
    }
}
```
Use wall-clock anchoring to avoid drift (Research Pitfall 1).

`startElapsedTicker(initialSeconds: Long)`:
```kotlin
private fun startElapsedTicker(initialSeconds: Long = 0L) {
    _elapsedSeconds.value = initialSeconds
    elapsedJob?.cancel()
    elapsedJob = viewModelScope.launch {
        while (true) {
            delay(1000L)
            _elapsedSeconds.value++
        }
    }
}
```

`updateRestState(restState: RestState)`:
```kotlin
private fun updateRestState(restState: RestState) {
    val current = _sessionState.value as? WorkoutSessionState.Active ?: return
    _sessionState.value = current.copy(restState = restState)
}
```

All methods that modify state run in `viewModelScope.launch` for Room suspend calls. All state updates go through `_sessionState.value = ...` to ensure atomic state transitions (Research Pitfall 5 -- single state flow).

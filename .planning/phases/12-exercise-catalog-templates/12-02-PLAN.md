```kotlin
    // This is THE approach for ExercisePicker -> TemplateEditor communication.
    // Do NOT use savedStateHandle or other alternatives.
    val parentEntry = remember(backStackEntry) {
        workoutNavController.getBackStackEntry<TemplateEditorRoute>()
    }
    val editorViewModel: TemplateEditorViewModel = koinViewModel(
        viewModelStoreOwner = parentEntry
    )
    ExercisePickerScreen(
        onExerciseSelected = { id, name, muscles ->
            editorViewModel.addExercise(id, name, muscles)
            workoutNavController.popBackStack()
        },
        navController = workoutNavController
    )
}
composable<WorkoutSessionRoute> { backStackEntry ->
    val route = backStackEntry.toRoute<WorkoutSessionRoute>()
    // WorkoutSession is Phase 13 -- show placeholder for now
    PlaceholderScreen(
        icon = Icons.Filled.FitnessCenter,
        title = "Workout Session",
        message = "Template: ${route.templateId}. Session screen coming in Phase 13."
    )
}
```

**IMPORTANT: ExercisePicker -> TemplateEditor wiring explanation:**
The `composable<ExercisePickerRoute>` block uses `workoutNavController.getBackStackEntry<TemplateEditorRoute>()` to get the back stack entry for the TemplateEditorRoute that is still on the stack (since ExercisePicker was navigated TO from the editor). It then obtains the same `TemplateEditorViewModel` instance from that entry's ViewModelStoreOwner via `koinViewModel(viewModelStoreOwner = parentEntry)`. When `onExerciseSelected` fires, it calls `editorViewModel.addExercise(id, name, muscles)` directly on the shared ViewModel instance, then pops back. The editor screen's `exercises` StateFlow automatically updates because it observes the same ViewModel.

If `getBackStackEntry<TemplateEditorRoute>()` does not compile with the type-safe route generics (some Navigation versions require string route), use: `workoutNavController.getBackStackEntry(TemplateEditorRoute.serializer().descriptor.serialName)` or the route class name. Verify which API is available in your Navigation Compose version.

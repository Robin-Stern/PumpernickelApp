**Composable signature:**
```kotlin
@Composable
fun ExerciseOverviewSheetContent(
    exercises: List<SessionExercise>,
    currentExerciseIndex: Int,
    onSelect: (Int) -> Unit,
    onMove: (from: Int, to: Int) -> Unit,
    onSkip: () -> Unit,
    onDismiss: () -> Unit
)
```

Note: This is the CONTENT composable, not wrapped in ModalBottomSheet. The ModalBottomSheet wrapper lives in WorkoutSessionScreen. This keeps the composable testable and reusable.

**Implementation:**

1. **Top bar:** Row with "Exercise Order" title (headlineSmall) and "Done" TextButton calling `onDismiss`

2. **LazyColumn body with sections:**

   a. **Completed section** — exercises before currentExerciseIndex that have at least one completed set:
   ```kotlin
   val completedIndices = (0 until currentExerciseIndex).filter { idx ->
       exercises[idx].sets.any { it.isCompleted }
   }
   ```
   - Section header: "Completed" in `MaterialTheme.typography.titleSmall`, uppercase, secondary color
   - Each row: `ListItem` with:
     - leadingContent: checkmark circle icon in green (`Icons.Default.CheckCircle`, tint `MaterialTheme.colorScheme.primary`)
     - headlineContent: exercise name in secondary/grey text
     - supportingContent: "{completed}/{targetSets} sets"
     - Modifier.clickable: `onSelect(index)`

   b. **Current section** — exercise at currentExerciseIndex:
   - Section header: "Current"
   - Single `ListItem` with:
     - leadingContent: play circle icon (`Icons.Default.PlayArrow`) in primary color
     - headlineContent: exercise name in bold (`FontWeight.SemiBold`)
     - supportingContent: "{completed}/{targetSets} sets"
     - trailingContent: if `currentExerciseIndex + 1 < exercises.size`, show "Skip" TextButton in orange (`Color(0xFFFF9800)`) calling `{ onSkip(); onDismiss() }`

   c. **Up Next section** — exercises after currentExerciseIndex:
   ```kotlin
   val pendingStart = currentExerciseIndex + 1
   val pendingExercises = if (pendingStart < exercises.size) {
       exercises.subList(pendingStart, exercises.size)
   } else emptyList()
   ```
   - Section header: "Up Next"
   - Each row: `ListItem` with:
     - leadingContent: circle outline icon (`Icons.Default.RadioButtonUnchecked`) in secondary
     - headlineContent: exercise name
     - supportingContent: "{completed}/{targetSets} sets"
     - trailingContent: Row of move-up/move-down IconButtons:
       - Move up: `Icons.Default.KeyboardArrowUp`, enabled when relativeIndex > 0
         - onClick: `onMove(relativeIndex, relativeIndex - 1)` (pending-relative)
         - Note: ViewModel.reorderExercise uses pending-relative from/to where pending starts at currentExerciseIndex + 1
       - Move down: `Icons.Default.KeyboardArrowDown`, enabled when relativeIndex < pendingExercises.size - 1
         - onClick: `onMove(relativeIndex, relativeIndex + 2)` (toOffset semantics matching SwiftUI .onMove — insert BEFORE destination index)
     - Modifier.clickable: `onSelect(pendingStart + relativeIndex)` (absolute index for jumpToExercise)

   Note on reorder semantics: The ViewModel's `reorderExercise(from, to)` uses SwiftUI onMove toOffset semantics where `to` is the insertion point. For move-up: `onMove(idx, idx - 1)`. For move-down: `onMove(idx, idx + 2)` (move past the item below). This matches the Phase 12 TemplateEditorScreen pattern.

3. **Section headers** are implemented as `item` blocks in LazyColumn with padding and styling:

   ```kotlin
   item {

   ```

4. **Dividers** between sections: `HorizontalDivider()` items
